AWS 的秘密：Anthropic 与 Bedrock 组合如何推动利润率攀升，让同行望尘莫及

2026 年 5 月，SemiAnalysis 发布重磅报告，深度解析 AWS 利润率逆势攀升的背后秘密。

当甲骨文（Oracle）和 Coreweave 因云业务利润低于预期让市场失望，Azure 的利润率持续走低，甚至连谷歌云的利润率也存在”虚高”嫌疑时，AWS 却在上个季度实现了息税前利润率环比增长 213 个基点。这一切背后的核心驱动力，正是 Anthropic 的爆发式增长与 Bedrock 平台的独特商业模式。

博客作者对 SemiAnalysis 文章做了解读与总结，并未完全直译原文。需要精确数据的读者请自行查阅原报告。

这份报告的核心结论十分明确：AWS 的利润率优势并非偶然，而是由差异化战略、独特的”代币即服务”（TaaS）商业模式和产能优势共同铸就的竞争壁垒。 更令人惊讶的是，这一成就建立在 AWS 服务器折旧年限仅 5 年（所有云服务商中最短）的基础之上——如果按其他云服务商的折旧标准计算，AWS 的实际利润率优势将更加显著。

一、云服务商的利润分化：只有 AWS 在真正崛起

过去几个季度，各大云服务提供商的运营利润率呈现出截然不同的走势：

  • Azure：利润率持续下行。微软严重依赖传统 IaaS 模式，而在 365 Copilot 和 GitHub Copilot 等 AI 领域始终未能形成真正的增长飞轮。
  • Oracle 和 Coreweave：云业务利润均低于市场预期，令投资者失望。
  • Google Cloud：近期利润率大幅攀升，但这幅图景暗藏玄机——我们稍后会详细揭示。
  • AWS：唯一一家利润率呈现真实上升趋势的云服务商，且这一成就是在最保守的 5 年服务器折旧政策下达成的。

本质上，AWS 已经找到了一个强大的利润增长引擎，而其他厂商仍在苦苦追赶。

二、AWS 的差异化战略四重奏

SemiAnalysis 认为，AWS 的成功源于四大差异化优势的组合：

2.1 风险偏好：敢于用资产负债表下注

AI 基础设施竞赛中，真正的赢家从不畏惧动用资产负债表。根据 SemiAnalysis 的数据中心行业模型，亚马逊已获得除谷歌之外所有云服务商中最多的电力资源——它比同行更早意识到：在算力受限的环境下，能源就是市场份额。这背后是数十亿美元的购电协议（PPA）和大胆的资本投入。

2.2 商业模式：唯一的”代币即服务”主导者

这是 AWS 最关键的差异化优势。亚马逊是唯一一家以”代币即服务”（Token-as-a-Service，简称 TaaS）作为 AI 业务主导模式的云服务商。其他厂商大多聚焦于多年期的 IaaS 租赁合同——这种模式虽然收入稳定，但利润率远不及 TaaS。

TaaS 的核心逻辑是：客户按使用的 Token 量付费，AWS 不仅赚取底层基础设施费用，还能获得收入分成。 这意味着每多卖出一个 Token，AWS 的利润率都在增厚。

2.3 规模与速度：无人能及的建站能力

SemiAnalysis 的数据中心模型显示，2025、2026 和 2027 年，没有任何云服务商的产能增长能超过 AWS。亚马逊不仅在电力采购上领先，其实际执行速度也远超同行。公司正在推出全新的模块化、预制化数据中心设计，这将进一步拉大其速度优势——在印第安纳州和密西西比州以闪电速度建设近 2GW 容量，而微软在威斯康星州的大规模 AI 集群建设则进展缓慢。

2.4 垂直整合：从 Graviton 到 Trainium

在 CPU 领域，亚马逊的第五代自研芯片 Graviton 已处于行业领先地位。在 AI 加速器市场，Trainium 芯片对推理和强化学习工作负载表现出极强的性价比优势。Trainium 如今已支撑超过 50% 的 Bedrock Token 使用量（AWS CEO Matt Garman，2025 年 11 月）。对于客户而言，Bedrock 底层硬件完全透明——他们只看到 Token，这意味着推理优化芯片与 Bedrock 天然契合。

三、Bedrock 深度解析：前沿模型的入口之争

3.1 API 端点市场：关键不在于”模型多”，而在于”模型好”

Bedrock 是 AWS 的一项服务，允许客户从众多选项中选择偏好的大语言模型（LLM），同时享受 AWS 的安全合规、统一计费和基础设施能力。

SemiAnalysis 指出，许多 API 端点提供商热衷于展示平台上的模型数量，但这恰恰掩盖了最关键的差异化因素：对前沿 LLM 的访问权限。Tokenomics 模型的数据证明，前沿模型贡献了 AI API 行业收入的绝大部分。

在这一点上，AWS、微软和谷歌拥有其他厂商无法企及的优势：

  • AWS：长期独占 Claude，近期又获得了 OpenAI 模型
  • 谷歌：同时拥有 Claude 和 Gemini
  • 微软：长期独占 OpenAI 模型，近期也获得了 Claude

目前，没有其他云服务商能够同时销售 Claude、OpenAI 和 Gemini 三种前沿模型的 Token。

3.2 TaaS 平台的经济学：IP 自有 vs 模型分发

TaaS 平台的商业模式取决于对模型 IP 的关系，分为两种：

IP 自有模式：与 AI 实验室的商业模式一致。固定成本来自基础设施（GPU 折旧、数据中心、电力等），收入取决于 Token 销量。利润率取决于定价水平与硬件利用率。

模型分发模式（Bedrock 的模式）：在此场景下，Anthropic 是实际卖方，将已售 Token 的全部收入入账。AWS 作为计算和市场平台提供商，既能获得基础设施费用（类似 EC2 的 IaaS 费用），还能获得分销或收入分成费用。后者正是提升利润率的关键——它使得在 Bedrock 上销售 Claude 成为 AWS 极具吸引力的业务。

对 AI 实验室而言，通过云服务商分发模型有两大优势：触达云服务商庞大的客户群体，以及无需签订昂贵的多年期 IaaS 合同即可获得算力。对云服务商而言，这意味着承担更多风险（无法靠照付不议合同锁定收入），但利润率显著更高。

3.3 Bedrock/Anthropic 交易结构

SemiAnalysis 对双方合作的理解包括三个组成部分：

  1. 固定的 IaaS 费用
  2. 收入分成
  3. 超额业绩门槛——当 Token/支出吞吐量超过特定水平时触发超额分成

SemiAnalysis 的 Tokenomics 模型详细建模了这一交易。数据显示，在 2026 年第一季度，Anthropic 在 Bedrock 上的收入达到每兆瓦约 2600 万美元，这意味着 Bedrock 的 EBIT 利润率约为 ~55%。

四、数据说话：Bedrock 如何重塑 AWS 的利润结构

4.1 AI 业务占比快速攀升

  • AWS 的 AI 收入占总收入比例，已从 2024 年 Q1 的 2% 增长至 2026 年 Q1 的 10%（作为对比：同期 GCP 为 36%，Azure 为 27%，但结构完全不同）
  • Bedrock 在 AWS AI 总收入中的占比，已从 2025 年 Q1 的 9% 稳步增长至如今的 37%
  • 相比之下，Azure 和 GCP 的 AI 业务中，AI IaaS 仍占 80% 以上

4.2 利润率贡献远超收入占比

这是最令人震撼的数据：尽管 Bedrock 仅占 AWS 总收入的 4%，却贡献了 AWS 毛利同比增长的 30%。2026 年 Q1 相对于 2025 年 Q1 的增量 EBIT 利润率，主要就是由 Bedrock 驱动的。

2026 年 Q2 的展望更加乐观：预计 Anthropic 每兆瓦总计算能力的 ARR 将达到 4200 万美元，Bedrock 收入占 AWS AI 收入的比重将升至 53%，为 AWS 总收入增长额外贡献 9 个百分点。

4.3 Bedrock 已经是一个收入巨兽

根据亚马逊财报电话会议披露，Bedrock 收入在 2026 年 Q1 和 2025 年 Q4 分别环比增长 170% 和 60%。SemiAnalysis 估算，Bedrock 目前年化收入已达 55 亿美元，其中 80%-90% 以上的客户使用 Anthropic 模型。

五、Anthropic 的爆发：Claude Code 点燃的增长引擎

AWS 的利润率故事离不开 Anthropic 的爆发式增长。2026 年 Q1，Anthropic 的表现堪称惊艳：

  • 单季度新增 210 亿美元净新增 ARR，总 ARR 达到 300 亿美元
  • 收入结构以 API 和企业端为主（与 OpenAI 偏重消费者的结构形成对比）
  • 核心驱动力是 Claude Code，这款产品在企业市场掀起热潮
  • 信用卡面板数据显示，Anthropic 在净新增客户中的市场份额已超过 OpenAI

更值得关注的是 Anthropic 利润率的惊人逆转：

  • 2024 年推理毛利率：-94%（巨额亏损）
  • 2025 年推理毛利率：38%
  • 2026 年当前推理毛利率：约 60% 中段

据《华尔街日报》2026 年 5 月 20 日报道，Anthropic 预计在调整股权激励后，将于 Q2 实现营业利润盈利。SemiAnalysis 的 Tokenomics 模型最新估算在收入、成本结构和 GAAP（未调整股权激励）EBIT 利润方面均高度吻合这一判断。

对于 AWS 而言，这意味着 Anthropic 不仅能带来收入规模，还能在利润率层面形成正向贡献。

六、产能竞赛：为什么 AWS 能遥遥领先？

在远超竞争对手的规模上部署 TaaS 平台，关键在于拥有更多计算能力。SemiAnalysis 的数据中心行业模型显示：

  • 2024-2026 年：只有微软的产能增长能与 AWS 接近
  • 2027 年：AWS 将完全占据压倒性优势

但总容量并非唯一相关因素。需按最终用户细分：

  • 微软：大量算力通过长期合同流向 OpenAI（仅 OpenAI 一家的积压订单就达到 Azure 全年营收的 2.5 倍），内部 AI 项目也消耗了大量容量，大幅减少了可供外部客户使用的算力
  • 谷歌：同时与 AWS 争云服务、与英伟达争硬件、与 Anthropic/OpenAI 争模型，是一家终极供应受限的企业

微软在 2024-2025 年经历了长达一年的数据中心建设暂停，严重压缩了其 2027 年的产能预期。追赶的唯一途径是向 Neocloud 大量租用容量，但成本高昂且会削弱利润率优势。

七、谷歌的”利润率幻觉”：被隐藏的训练成本

对于”谷歌云利润率也在上升”的质疑，SemiAnalysis 给出了尖锐的反驳：

谷歌云的利润率本质上是”EBTIT”利润率——Earnings Before Training, Interest and Taxes（扣除训练成本、利息和税之前的利润）。

DeepMind/Gemini 的训练支出被计入”Alphabet 层面活动”而非 GCP 板块。根据谷歌最新 10-Q 文件，这一科目在 2026 年 Q1 达到 54 亿美元（上年同期为 30 亿美元）。所有 Gemini API 收入都以高于平均水平的利润率流入谷歌云，从而美化了 GCP 的利润率表现，而超过 100 亿美元的经常性训练成本却被归入别处。

与此形成对比的是，AWS 的利润率数字是真实的——所有成本都在 AWS 板块内如实反映，且采用最保守的 5 年折旧政策。

八、启示与展望

SemiAnalysis 对云服务商和 AI 实验室的未来做出以下判断：

短期（2-3 年）：领先的 IaaS 云服务商利润率将持续改善。但在产能大幅扩张和资本支出通胀期仍能实现利润率上升的，唯有 AWS——这得益于其团队的战略决策。

长期趋势：

  • AI 实验室将日益垂直化其推理栈，挤压纯 IaaS 供应商的利润率
  • 云服务商将通过定制芯片和战略合作伙伴关系实现垂直整合，利用庞大的多元化客户基础创造价值
  • TaaS 业务已是 100 亿美元级别的 ARR 市场，而 Neocloud 和 Oracle 在这方面几乎为零

亚马逊已经找到了自己的 AI 公式：用 Bedrock 做分发平台、用 Anthropic 做模型供应商、用 Trainium 做底层芯片、用 TaaS 做商业模式、用超强产能做护城河。 这五个要素的协同效应，正在将 AWS 与竞争对手之间的差距越拉越大。
